## 1. ホーム・本人確認・マッチ成立（画像: 18.31.17）

### 1.1 スプラッシュ / ロゴ

- 背景: オフホワイト
- 中央: 赤い「L」ロゴ + 「Lobby」

### 1.2 本人確認のお願い

**見出し・本文（要約）**

- タイトル: 「本人確認のお願い」
- 安全のため身分証提出が必要、などの説明文

**操作**

| UI | 文言 |
|----|------|
| 書類種別ボタン（赤・白文字・右矢印） | 運転免許証 / マイナンバーカード / パスポート |
| アップロード枠 | アップロード画像（1）（2）— グレー枠 + カメラ/ギャラリーアイコン |
| 送信 | 「送信する」（紺/ダークブルー系の広いボタン） |

**送信後**

- 「確認書類を送信しました」— 審査中（3 営業日以内など）
- 中央ボックス: 「審査中」（赤枠・赤文字）

**モーダル**

| 種別 | 文言 |
|------|------|
| 成功 | 「本人確認が完了しました！」→ ボタン「Lobbyに参加する」（赤） |
| 失敗 | 「本人確認を再実行してください」+ 理由（例: 画像が不鮮明）→「本人確認画面に戻る」（紺） |

### 1.3 ホーム画面

**ヘッダー（赤・白文字）**

- 地名例: 「名古屋」
- 「Season 1」
- バッジ例: 「シーズン終了まで残り 3日」

**メインカード（白・角丸）**

- Lobby ロゴ、シーズン名、日付
- 大きな数字例: **62**（参加者番号イメージ）
- 下に英数字コード（QR 用コード表示想定）

**アクション（カード下・横並び）**

| ボタン | 文言 |
|--------|------|
| QR 表示 | 「表示する」+ QR アイコン |
| スキャン | 「スキャン」+ スキャナーアイコン |

### 1.4 QR 表示

- カード内に大きな QR（赤白）
- 名前例: 「あおい」+ ID コード
- 「戻る」リンク

### 1.5 マッチングしました（フルスクリーン）

- 背景: オレンジ→赤グラデ
- 2 つの丸アバター + 中央の鍵アイコン + 白線で接続
- 文言: 「マッチングしました」「お相手の情報を見てみましょう」等
- ボタン: 「お相手の情報を見る」（白地・赤文字）
- 下: 「×」閉じる

### 1.6 スキャン

- カメラビューファインダー
- 左上: 戻る

### 1.x このブロック — 確認用

**変えない（現行）**

- ホーム QR / スキャン / コード入力、No.001〜 採番、`season-config` カウントダウン
- 本人確認の**データモデル**（Storage + `identityStatus`）。書類種別 3 ボタンは**未実装なら要判断**

**変える（デザイン参考）**

- マッチ成立の**全画面演出**（現状は別 UI の可能性 → 要確認）
- 本人確認 UI の**見た目**（種別選択・2 枚アップロード）— 機能を増やす場合は仕様化が必要

**要確認**

- ホームの巨大数字「62」= `participantNo` か別物か
- 書類種別 3 択 + 画像 2 枚を**本当に採用するか**（現行は単一アップロード想定）

## 2. マッチング履歴（画像: 18.31.28）

### 2.1 一覧（マッチあり）

**ヘッダー**

- タイトル（赤）: 「マッチング履歴」

**シーズン帯（赤）**

- 「あなたが参加中のシーズン」

**シーズン情報**

- 「2026年名古屋シーズン」
- 「このシーズンには60人が参加しています」

**一覧セクション**

- 見出し: 「これまでにマッチングした一覧」+ ヘルプ **?** アイコン

**グリッド（3 列）**

- 丸いプロフィールプレースホルダー
- 右下バッジに**数字**（マッチ回数ではなく、デザイン説明では「キーホルダー色＝マッチした回数」）

  - **橙**: 2 回
  - **赤**: 3 回以上
- 例示数字: 7, 23, 31, 33, 38, 41, 44, 51 等（凡例用）

### 2.2 ヘルプモーダル「マッチング一覧について」

- キーホルダー（バッジ）の色の意味（黄/橙/赤）
- 複数ブロック「ここにテキストが入ります」
- 下: 丸い **×** で閉じる

### 2.3 空状態

- 「現在、マッチングしていません」

### 2.4 相手詳細「マッチングした相手」

**ヘッダー**

- 戻る + タイトル「マッチングした相手」+ **×**

**プロフィール**

- 大きなアバター
- 名前: **あおい**
- ID: `@20260329-007`（※実装では **No.xxx** に置換）
- 「マッチングした回数 **1回**」

**シンクロ**

- 見出し: 「あなたとのシンクロ割合」
- サブ: 「12問中8問 意見が一致しました」
- **大きな円形ゲージ 66%**（中央に数字）

**12 問の回答一覧**

- Q1: 「旅行に行くなら？」→「事前にしっかり計画を立てたい」
- Q2〜Q12: 「ここに回答」プレースホルダ
- 各回答の左に**チェックマーク絵文字**（※実装では**使わない**）

### 2.5 自由メモモーダル

- タイトル: 「自由メモ (自分だけに表示)」
- 入力: 「ニックネーム」「自由メモ欄」
- ボタン: 「保存」（赤枠）

**要確認**

## 3. イベントカレンダー・お知らせ（画像: 18.32.04）

### 3.1 イベントカレンダー

**ヘッダー**

- タイトル（赤）: 「イベントカレンダー」
- 左: ベルアイコン
- 赤文字: 「シーズン終了まで残り 32日」+ プレースホルダ一行

**カレンダー**

- 「2025年 12月」+ 前後矢印
- グリッド。選択日例: **3**（赤丸）

**下部パネル（日付選択時）**

- ヘッダー: 「2025年12月3日 水」+ **×**
- **タブ 3 つ:**
  - 朝イベント（オレンジの太陽）
  - 昼イベント（黄の太陽）
  - 夕イベント（紫の月/星）
- **夕タブ選択時:** 「Event Name」+ 説明プレースホルダ複数行
- **朝タブ・空:** 「開催はありません」+ 他の日を見る促し
- パネル閉じた状態 = カレンダーのみ

### 3.2 お知らせ（独立画面）

**ヘッダー**

- 「お知らせ」（赤）

**リスト（日付グループ）**

- 日付例: 「2025年12月2日(火)」
- 各項目: 左グレー丸、送信者/カテゴリ + 時刻、**太字タイトル**、本文プレースホルダ
- タイトル例:
  - 「本日は〇〇組がマッチング！」
  - 「12月24日にイベント開催！」

### 3.x 確認用

**変えない**

- お知らせは**独立タブにしない**（ホーム折りたたみ + 未読バッジ）
- イベントの **朝/昼/夕** 枠・Firestore モデル（現行 `EventSlotSection` 系）

**変える**

- カレンダー UI の**見た目**（タブアイコン色、下部シート）
- シーズン残り日数の**帯の見せ方**（文言は `season-config` 優先）

**要確認**

- デザインの「お知らせタブ」は**採用しない**で合意済みか → Cursor 優先で **不採用**

## 4. マイページ・チャット（画像: 18.32.13）

### 4.1 マイページ

（プロフィール編集・各種設定は **セクション 5** と重複）

- ヘッダー: 「マイページ」
- カバー + 丸アバター + 編集鉛筆
- 名前「あおい」+ `@…`（※ **No.xxx**）

- メニュー:
  - 「あなたのマッチング履歴」— 赤サブ「新たに一人とマッチングしました」
  - 「各種設定」
  - 「お問い合わせ・ヘルプ」

※スクショのボトム 6 タブ（お知らせ・チャットバッジ 3）は**採用しない**。

### 4.2 チャット会話

**ヘッダー（赤）**

- 左: ×
- 中央: 「期限：2025年12月4日 23:59まで」
- 右: ハンボーガー（設定）

**本文（ベージュ）**

- 日付区切り: 「2025年12月2日(火)」
- 受信: 左アバター + 白吹き出し
- 送信: 右白吹き出し
- 下: 「2025/12/02, 12:00 既読」/「未読」

**入力欄**

- 左: 画像アイコン
- 中央プレースホルダ: 「ここにニックネームを入力」（※おそらくメッセージ入力の誤記）
- 右: 送信（紙飛行機）

### 4.3 チャット設定（ボトムシート）

- タイトル: 「設定」
- 項目:
  1. プロフィールを見る
  2. **ブロックする**（禁止アイコン）
  3. DM通知設定
  4. **ユーザー通報**（赤文字）

### 4.x 確認用

**変えない**

- チャット 1:1・期限表示・**通報のみ**（ブロックなし）
- 相性％: 名前横（一覧 + 会話サブヘッダー）— 済
- 会話ヘッダーは**期限が主**（デザインどおり赤帯）

**変える**

- 吹き出し・既読表示の**スタイル**
- 「プロフィールを見る」→ 履歴詳細 / 相手プロフィールへの導線

**要確認**

- **ブロック・DM通知設定**は実装するか（現仕様: **しない**）
- 入力プレースホルダは「メッセージを入力」等にするか
